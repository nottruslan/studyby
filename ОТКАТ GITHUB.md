# ОТКАТ GITHUB

Команды для работы с версиями и откатом.

---

## Посмотреть историю версий

```bash
git log --oneline
```
Показывает список коммитов и их короткие хэши. Хэш нужен для отката к конкретной версии.

---

## Откат на одну версию назад

```bash
git reset --hard HEAD~1
```

## Откат на две версии назад

```bash
git reset --hard HEAD~2
```

## Откат на N версий назад

```bash
git reset --hard HEAD~N
```
(подставь число вместо N)

## Откат к конкретной версии по хэшу

```bash
git reset --hard ХЭШ
```
Пример: `git reset --hard 6ada34c`

---

## Откат только одного файла

```bash
git checkout ХЭШ -- путь/к/файлу
```
Пример: `git checkout 6ada34c -- README.md`

---

## Откат последнего коммита, но оставить изменения в файлах

(коммит отменяется, правки остаются — можно заново отредактировать и закоммитить)

```bash
git reset --soft HEAD~1
```

---

## После отката — обновить GitHub

Если ты уже пушил и сделал откат локально, на GitHub всё ещё старая история. Чтобы и там была откатанная версия:

```bash
git push --force
```
Внимание: перезаписывает историю на GitHub. Не делай, если репо общий и другие пушат в ту же ветку.

---

## Сохранить версию и отправить на GitHub

```bash
git add .
git commit -m "описание изменений"
git push
```

---

## Проверить/поменять адрес репозитория

Посмотреть текущий remote:
```bash
git remote -v
```

Поменять URL (если создал другой репо или опечатка):
```bash
git remote set-url origin https://github.com/ЛОГИН/ИМЯ-РЕПО.git
```

---

## Сбросить сохранённый логин GitHub (если «Repository not found»)

```bash
git credential-osxkeychain erase
host=github.com
protocol=https
```
(ввести три строки, дважды Enter). При следующем `git push` запросит вход заново.
