# Studby Coding

Проект под версионирование через Git и GitHub.

---

## По шагам: первый раз подключить GitHub

**Шаг 1.** Зайди на https://github.com и войди в аккаунт (или зарегистрируйся).

**Шаг 2.** Нажми зелёную кнопку **New** (или открой https://github.com/new).

**Шаг 3.** В поле **Repository name** введи, например: `studby-coding`.  
Остальное не трогай: **не** ставь галочки «Add a README» и «Add .gitignore». Нажми **Create repository**.

**Шаг 4.** Открой терминал (в Cursor: Terminal → New Terminal) и перейди в папку проекта:
```bash
cd "/Users/ru/Desktop/Studby Coding"
```

**Шаг 5.** Привяжи свой репозиторий (подставь вместо `ТВОЙ_ЛОГИН` логин с GitHub):
```bash
git remote add origin https://github.com/ТВОЙ_ЛОГИН/studby-coding.git
```

**Шаг 6.** Отправь код на GitHub:
```bash
git push -u origin main
```
Если спросит логин/пароль — войди через браузер или используй токен. После этого проект будет на GitHub.

---

## По шагам: как сохранять версии и пушить

**Шаг 1.** После изменений в проекте открой терминал в папке проекта.

**Шаг 2.** Добавь все изменения:
```bash
git add .
```

**Шаг 3.** Сделай «снимок» с коротким описанием:
```bash
git commit -m "что сделал, например: добавлена главная страница"
```

**Шаг 4.** Отправь на GitHub:
```bash
git push
```

Так у тебя будут версии, к которым можно откатиться.

---

## По шагам: откатиться на прошлую версию

**Шаг 1.** Посмотри список версий (коммитов):
```bash
git log --oneline
```
Скопируй хэш нужной версии (первые 6–7 символов, например `6ada34c`).

**Шаг 2.** Откатить весь проект к этой версии (файлы станут как тогда):
```bash
git reset --hard ХЭШ
```
Пример: `git reset --hard 6ada34c`

**Шаг 3.** Если нужно, чтобы и на GitHub была эта версия (осторожно, перезаписывает историю):
```bash
git push --force
```

Откат только одного файла:
```bash
git checkout ХЭШ -- путь/к/файлу
```
Пример: `git checkout 6ada34c -- index.html`
